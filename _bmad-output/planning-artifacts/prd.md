# Micro-PRD: carlossimpore.com (Cyber-Architect Portfolio)

## 1. Project Goal
Build a high-performance, AI-automated professional portfolio that functions as a living "Proof of Concept" platform for "Titan-grade" engineering resilience and AI-driven automation.

## 2. Target Audience
- **Overwhelmed Solopreneurs**: Seeking scalable automation and reliable systems.
- **Visionary Tech Leads**: Seeking strategic technical collaboration and high-fidelity engineering.

## 3. Core Functional Requirements (Pages)

### 3.1 Home (Hero Engine Core)
- High-impact visual landing showcasing the "Hybrid-Engine" value proposition.
- Immediate understanding of value within 10 seconds.

### 3.2 Autonomy Lab
- Interactive grid/gallery of autonomous system proofs.
- "Lab Waves" aesthetic.

### 3.3 Decision Log
- Public log of architectural and strategic decisions (Transparency as a feature).
- Technical reliability demonstration.

### 3.4 Resilience Story
- Narrative explaining "Titan Resilience" (Offline-first, deep engineering).

### 3.5 AI Briefing Architect
- Interactive, streaming chat-like interface for strategic briefings.
- Direct lead generation through automated strategic analysis.

### 3.6 Case Studies & Service Catalog
- Multi-domain showcases (e.g., Treasurer app).
- Clear ROI-focused service offerings.

## 4. Technical Requirements (Titan Resilience)

### 4.1 Frontend Architecture
- **Framework**: Next.js 14+ (App Router).
- **Styling**: Vanilla CSS / Tailwind (to match Cyber-Architect design system).
- **Aesthetic**: Obsidian Black, Neon Blue (#3B82F6), Emerald (#10B981). Glassmorphism and dynamic micro-animations.

### 4.2 Local-First & Event-Sourcing
- **Offline Capability**: Site must remain functional and "smart" even without internet connection.
- **State Management**: Event-sourced data layer to ensure record integrity and auditability.
- **Synchronization**: Bi-directional sync strategy for real-time data persistence.

### 4.3 AI Engine Bridge
- **Pattern**: Clean integration of 'AI Briefing Architect' agent.
- **Mechanism**: Streaming responses via Server Actions.
- **Context Handling**: Efficient injection of domain-specific briefings into the agent stream.

### 4.4 Modular Multi-Domain Structure
- Structure must support evolving modules: `Autonomy Lab`, `Decision Log`, `Case Studies`.
- Domain-driven folder organization for scalability.
